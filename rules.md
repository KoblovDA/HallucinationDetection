# Rules

Правила взаимодействия с проектом. Эти правила должны соблюдаться на каждой итерации.

## Коммуникация

- Общение с пользователем — **на русском**.
- Финальный Jupyter Notebook — **на английском**, стиль «как написал человек», без типичных LLM-маркеров («Let's dive into…», «In this section we will…», избытка emoji, перегруженного markdown с буллетами на каждое предложение).
- Перед нетривиальной реализацией — согласовать подход с пользователем, не «убегать вперёд».

## Файлы проекта

В корне проекта поддерживаются три «живых» документа:

| Файл | Что в нём |
|---|---|
| `description.md` | Подробное описание задачи + краткие выжимки из статей. Это **контекст**, который всегда полезно иметь под рукой. Обновлять при изменении понимания задачи. |
| `tasks.md` | Все задачи проекта — текущие, выполненные, запланированные. Обновлять по ходу. |
| `insights.md` | Инсайты по экспериментам: что сработало, что нет, почему. Это пригодится для секции Discussion финального отчёта. |
| `rules.md` | Этот файл — правила работы. |

Эти файлы не являются частью финальной сдачи; они служат рабочим контекстом.

## Финальный артефакт

- Итог сдачи — один Jupyter Notebook (`.ipynb`) на английском, структура по `assignment_template.ipynb`.
- **Не редактировать `.ipynb` напрямую** — слишком много токенов уходит на JSON-оверхед.
- Вместо этого: держать исходники как набор `.py` (код) и `.md` (markdown-ячейки), а финальный `.ipynb` собирать скриптом в самом конце.
- **Не пересобирать `.ipynb` на каждой итерации** — только когда готов финал.

## Код и эксперименты

- Все эксперименты — Python 3, желательно воспроизводимые в Google Colab или Kaggle (free tier, ~16 GB VRAM).
- Если метод требует крупной LLM (например, LookBackLens с Llama-2-7B), считать фичи отдельно на сервере и класть в ноутбук уже предвычисленные результаты + скрипт извлечения для воспроизводимости.
- Использовать `requirements.txt` или явные `pip install` в первой ячейке.
- Все случайные seed'ы фиксировать (для воспроизводимости).

## Доступные ресурсы

- Free Colab + Kaggle (для финального ноутбука).
- OpenRouter API (платный, есть бюджет) — для синтетической инжекции, LLM-as-judge.
- Личный сервер — для крупных моделей, не влезающих в Colab.

## Память Claude

Параллельно ведётся файловая память Claude (`~/.claude/projects/.../memory/`):
- `MEMORY.md` — индекс,
- отдельные `*.md` файлы с типами: `project`, `user`, `feedback`, `reference`.

Память хранит вещи, которые **не выводятся из текущего состояния файлов проекта**: предпочтения пользователя, мотивации решений, история неочевидных выборов. Память не дублирует `description.md` / `tasks.md` / `insights.md`.
